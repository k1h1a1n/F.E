Password hashing is a process of turning a password into a scrambled representation of a password, Let’s say Arjun encrypts his file with password “hunter2”.

Durity uses the hash function to convert his passwords in a hashed format. The hash of “hunter2”, which is “2ab96390c7dbe3439de74d0c9b0b1767”, is then stored in his local device.

Now when Arjun tries to decrypt his file, he types “hunter2” into the password field.

Durity puts the password through the hash function again, compares the current hash value and locally stored hash value yielding the same hash string and letting him log in successfully.

If Arjun had typed in “hunter1”, the result of the function would be “726ad07bc398372b56a52e3de8693679”.

He would not be able to decrypt the file. If the hashes match the user is granted access. If not, the user is told they entered invalid login credentials.
